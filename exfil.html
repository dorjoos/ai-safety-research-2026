Securing Your Copilot Integration

When deploying Microsoft Copilot in enterprise environments, follow these security best practices to minimize risk from prompt injection and data leakage attacks.

1. Authentication and Access Control

Copilot uses Auth0 for authentication with the following default scopes:

openid profile email ChatAI.ReadWrite offline_access

Ensure that your organization's conditional access policies apply to Copilot sessions. The offline_access scope grants refresh tokens, which should be monitored for unusual usage patterns.

2. Connector Security

When connecting external services (Google Drive, Gmail, etc.), Copilot receives OAuth tokens that grant read access to user data. Implement the following controls:

  • Audit connector usage via the /c/api/user/settings endpoint
  • Monitor allowOneDriveAccess and allowOutlookAccess toggles
  • Review connected service permissions quarterly
Warning: Connected services expose data to Copilot's processing pipeline. Documents accessed via connectors may be included in Copilot's context window, making them susceptible to indirect prompt injection attacks.

3. Deep Research Security

The "deep research" feature allows Copilot to browse and synthesize information from multiple web sources. This creates an indirect prompt injection attack surface:

  1. Copilot visits web pages containing hidden instructions
  2. Hidden text is ingested into the model's context
  3. The model may follow injected instructions, leading to data leakage

Mitigate this by:

  • Restricting deep research to trusted domains via web filtering
  • Monitoring Copilot responses for signs of injection (unexpected formatting, image tags, links)
  • Implementing output monitoring that flags responses containing URLs not present in the original query

4. Incident Response

If you suspect a prompt injection attack against your Copilot deployment:

  1. Review the conversation history via /c/api/library/recent
  2. Check for unauthorized connector activations in user settings
  3. Audit WebSocket messages at wss://copilot.microsoft.com/c/api/chat
  4. Report the incident to MSRC at msrc.microsoft.com
